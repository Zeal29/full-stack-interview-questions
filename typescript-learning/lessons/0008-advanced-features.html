Advanced Type Features

Conditional types, infer, as const, and template literal types

What You'll Learn

  • Write conditional types with extends
  • Use infer to extract types
  • Apply as const for narrow type inference
  • Create template literal types for string manipulation

Why Advanced Types?

TypeScript's type system is Turing-complete — you can perform computations at the type level. Conditional types let you choose types dynamically. The infer keyword extracts types from patterns. as const locks down literal types. Template literal types let you manipulate strings at the type level. Together, these features let you build type-safe APIs that guide users and catch errors before runtime.

You won't use these every day, but libraries like Redux, Zod, and React Router rely heavily on them. Understanding these features helps you read library types and write smarter abstractions.

1 What are conditional types in TypeScript?

Conditional types let you choose a type based on a condition, using the syntax T extends U ? X : Y. If T is assignable to U, the type resolves to X; otherwise it resolves to Y. Think of it as a ternary operator for types.

The infer keyword works inside conditional types to extract and capture a type from a pattern. It's like a regex capture group — you declare a type variable that TS fills in during checking. 

// Basic conditional type
type IsString<T> = T extends string ? true : false;

// Using infer to extract return type
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

type A = ReturnType<() => string>;  // string
type B = ReturnType<(x: number) => void>; // void
💡 Memory Hook: Conditional types = ternary for types. infer = "extract and capture a type" — like a regex capture group but for type positions.

2 How does infer work in conditional types?

infer extracts and captures a type within a conditional. You place it inside the extends clause of a conditional type to declare a type variable that TypeScript will infer from the matched pattern. It only works inside extends conditionals — you can't use it standalone.

Common uses: extracting array element types, function return types, promise unwrapped types, and tuple element types. 

// Extract element type from an array
type ArrayElement<T> = T extends (infer E)[] ? E : never;

type Num = ArrayElement<number[]>; // number
type Str = ArrayElement<string[]>; // string

// Unwrap Promise type
type Awaited<T> = T extends Promise<infer U> ? U : T;

type Result = Awaited<Promise<string>>; // string
💡 Memory Hook: infer = "extract this type for me". Like a regex capture group for types — it only works inside extends conditionals.

3 What does as const do in TypeScript?

as const makes TypeScript infer the narrowest possible literal types. It converts every property to readonly with literal types — strings become their exact string literal, numbers become their exact numeric literal, and arrays become readonly tuples.

Without as const, TypeScript widens types: "hello" becomes string, 42 becomes number. With as const, they stay as "hello" and 42. Essential for building discriminated unions and exact tuple types. 

// Without as const — types are widened
const config = { host: "localhost", port: 3000 };
// { host: string; port: number }

// With as const — literal types, locked down
const config = { host: "localhost", port: 3000 } as const;
// { readonly host: "localhost"; readonly port: 3000 }

// Tuples become readonly
const pair = ["a", 1] as const;
// readonly ["a", 1] — a readonly tuple, not string[]
💡 Memory Hook: as const = "lock it down". Widening stops, mutability stops — the type becomes exactly what you wrote.

4 What are template literal types in TypeScript?

Template literal types let you perform string manipulation at the type level. Using backticks in a type position (just like template literals in JS), you can generate and transform string union types. TypeScript includes four built-in intrinsic types: Uppercase, Lowercase, Capitalize, and Uncapitalize. 

// Generate string unions from other unions
type Direction = "top" | "bottom" | "left" | "right";
type CSSMargin = `margin-${Direction}`;
// "margin-top" | "margin-bottom" | "margin-left" | "margin-right"

// Built-in string manipulation types
type Upper = Uppercase<"hello">;       // "HELLO"
type Lower = Lowercase<"HELLO">;       // "hello"
type Cap   = Capitalize<"hello">;     // "Hello"
type Uncap = Uncapitalize<"Hello">; // "hello"

// Combine with unions for event names
type EventName = `on${Capitalize<"click" | "focus">}`;
// "onClick" | "onFocus"
💡 Memory Hook: Template literal types = template literals for types. Same backtick syntax as JS, but at the type level — generates string unions automatically.

🧪 Quick Check

1. What does T extends U ? X : Y represent in TypeScript?

✅ Correct! Conditional types use ternary syntax to choose between two types based on an extends check.
❌ This is a conditional type — TypeScript checks if T extends U at compile time and resolves to X or Y accordingly.

2. What does as const do to an object literal?

✅ Correct! as const locks everything down — readonly properties with exact literal types instead of widened ones.
❌ as const makes properties readonly and infers the narrowest literal types (e.g., "hello" instead of string).

3. Where can the infer keyword be used?

✅ Correct! infer only works inside the extends clause of a conditional type — it can't be used anywhere else.
❌ infer is restricted to the extends clause of conditional types. It captures a type from the pattern being matched.
