Utility Types Reference Table

TypeScript ships with a set of built-in utility types — generic types that transform other types. They are implemented as mapped types under the hood. Here's the complete cheat sheet:

Utility Type What It Does Example
Partial<T> Makes all properties optional Partial<User>
Required<T> Makes all properties required Required<User>
Readonly<T> Makes all properties read-only Readonly<Config>
Pick<T, K> Selects a subset of properties Pick<User, "email" | "name">
Omit<T, K> Removes a subset of properties Omit<User, "password">
Record<K, V> Constructs an object type with keys K and values V Record<string, number>
Exclude<U, E> Removes types from a union Exclude<"a"|"b"|"c", "a">
Extract<U, E> Extracts types from a union Extract<string|number, number>
ReturnType<F> Extracts the return type of a function ReturnType<typeof fn>
Parameters<F> Extracts parameter types as a tuple Parameters<typeof fn>

1 What are mapped types and how do they work?

A mapped type creates a new type by transforming each property of an existing type. You iterate over a union of keys using in keyof and apply a transformation to each property's type or modifiers. 

// Make all properties readonly
type MyReadonly<T> = {
  readonly [K in keyof T]: T[K];
};

// Make all properties optional
type MyPartial<T> = {
  [K in keyof T]?: T[K];
};

// Usage
interface User {
  name: string;
  age: number;
}

type ReadonlyUser = MyReadonly<User>;
// { readonly name: string; readonly age: number }

Key modifiers: You can add readonly, ? (optional), or remove them with -readonly or -?. Mapped types are the foundation of all built-in utility types like Partial, Readonly, Required, and Pick.

💡 Memory Hook: "Loop over keys and transform." Think of it as a for-loop for types — [K in keyof T] iterates each key, and you decide what happens to each property.

2 What are the most commonly used utility types in TypeScript?

TypeScript ships with built-in utility types that transform existing types. The most commonly used ones: 

interface User {
  name: string;
  email: string;
  age: number;
  password: string;
}

// Partial — all properties optional (edit mode)
type UserUpdate = Partial<User>;
// { name?: string; email?: string; age?: number; password?: string }

// Required — all properties required (undo optional)
type StrictUser = Required<User>;

// Readonly — cannot reassign properties
type FrozenUser = Readonly<User>;

// Record — typed dictionary
type UserMap = Record<string, User>;
// { [key: string]: User }

// Exclude / Extract — union manipulation
type Status = "active" | "inactive" | "banned";
type ActiveStatus = Exclude<Status, "banned">;
// "active" | "inactive"

// Parameters — extract function params as tuple
function createUser(name: string, age: number) { /* ... */ }
type CreateUserArgs = Parameters<typeof createUser>;
// [string, number]
💡 Memory Hook: Partial = edit mode (patch updates). Pick = select columns (like SQL SELECT). Omit = remove columns (like SQL exclude). Record = typed dictionary. Memoize these — they are used everywhere in real codebases.

3 What is the difference between Omit and Pick?

Both create a subtype by selecting a subset of properties, but they take opposite approaches:

  • Omit<T, K> — removes the specified keys. "Everything except these."
  • Pick<T, K> — selects only the specified keys. "Only these."
interface User {
  id: number;
  name: string;
  email: string;
  password: string;
  role: string;
}

// Omit — remove password (keep everything else)
type PublicUser = Omit<User, "password">;
// { id: number; name: string; email: string; role: string }

// Pick — select only login fields
type LoginPayload = Pick<User, "email" | "password">;
// { email: string; password: string }

When to use which? If you want most properties and need to exclude a few, use Omit. If you want only a few properties from a large type, use Pick. They are inverses of each other — Omit<T, K> ≈ Pick<T, Exclude<keyof T, K>>.

💡 Memory Hook: Omit = "everything except." Pick = "only these." Think of Omit as a blacklist and Pick as a whitelist for properties.

4 How does ReturnType work and when would you use it?

ReturnType<F> extracts the return type of a function type. You pass it typeof functionName (not the function itself) to get the type the function returns — without calling it. 

function getUser() {
  return { id: 1, name: "Ali", email: "ali@mail.com" };
}

// Extract the return type without calling the function
type UserType = ReturnType<typeof getUser>;
// { id: number; name: string; email: string }

// Useful for async functions too
async function fetchUser() {
  const res = await fetch("/api/user");
  return res.json() as UserType;
}

// Unwraps the Promise — gives you UserType, not Promise<UserType>
type ResolvedUser = ReturnType<typeof fetchUser>;
// Promise<UserType> — use Awaited<...> to unwrap fully
type FullyResolved = Awaited<ReturnType<typeof fetchUser>>;
// UserType

Common pattern: When a function returns a complex object and you need that shape elsewhere (e.g., for props, state, or another function's parameter), use ReturnType to avoid duplicating the type definition.

💡 Memory Hook: "What does this function return?" — ReturnType<typeof fn> answers that question. Always pair with typeof since you need the function's type, not the function itself.

🧪 Quick Check

1. Which utility type makes all properties of an object optional?

✅ Correct! Partial<T> makes every property in T optional — perfect for patch/update operations.
❌ Partial<T> is the utility type that makes all properties optional. Required does the opposite, Readonly prevents reassignment, and Omit removes properties.

2. Given interface User { id: number; name: string; email: string; password: string; }, which type produces { email: string; password: string; }?

✅ Correct! Both Pick<User, "email" | "password"> and Omit<User, "id" | "name"> produce the same result — Pick selects the specified keys, Omit removes the specified keys.
❌ The answer is both A and B. Pick selects only "email" and "password", while Omit removes "id" and "name" — both yield { email: string; password: string; }. They are inverse operations.

3. How do you extract the return type of a function named getUser?

✅ Correct! ReturnType expects a function type, so you use typeof getUser to get the type of the function.
❌ ReturnType takes a type, not a value. Use typeof getUser to reference the function's type: ReturnType<typeof getUser>.
