Type Narrowing & Guards

How TypeScript refines types within conditional blocks

What You'll Learn

  • Write custom type guards with type predicates

Type Narrowing Techniques

TypeScript starts with a wide type (like string | number) and narrows it within conditional blocks based on checks you perform. The compiler tracks these checks and refines the type accordingly — this is called control flow based type analysis.

1. typeof — Primitive narrowing

Works for string, number, boolean, symbol, bigint, undefined, object, function.

function printValue(value: string | number) {
  if (typeof value === "string") {
    // TypeScript knows: value is string
    console.log(value.toUpperCase()); // ✅ OK
  } else {
    // TypeScript knows: value is number
    console.log(value.toFixed(2)); // ✅ OK
  }
}

2. instanceof — Class/constructor narrowing

Checks the prototype chain. Essential for narrowing Error subtypes, Date, RegExp, Array, and custom classes.

function handleError(err: Error | string) {
  if (err instanceof Error) {
    console.log(err.stack); // ✅ err is Error
  } else {
    console.log(err.toUpperCase()); // ✅ err is string
  }
}

3. in operator — Property-check narrowing

Checks whether a property exists on an object. Useful for discriminating objects with different shapes.

type Fish = { swim: () => void };
type Bird = { fly: () => void };

function move(animal: Fish | Bird) {
  if ("swim" in animal) {
    animal.swim(); // ✅ animal is Fish
  } else {
    animal.fly(); // ✅ animal is Bird
  }
}

4. Truthiness narrowing & equality checks

null/undefined elimination via truthiness checks, plus strict equality for literal types.

function printName(name: string | null) {
  if (name !== null) {
    console.log(name.toUpperCase()); // ✅ name is string
  }
}

function handle(val: "a" | "b") {
  if (val === "a") {
    // val is "a"
  } else {
    // val is "b"
  }
}

5. Discriminated unions — The state machine pattern

Use a shared literal property (the discriminant) to tag each union member. TypeScript narrows based on that field in switch/if.

type Shape =
  | { kind: "circle"; radius: number }
  | { kind: "rect"; w: number; h: number };

function area(s: Shape): number {
  switch (s.kind) {
    case "circle":
      return Math.PI * s.radius ** 2; // ✅ s.radius exists
    case "rect":
      return s.w * s.h; // ✅ s.w and s.h exist
  }
}

6. Custom type guards — Type predicates

When built-in checks aren't enough, write a function with a type predicate return type (x is Type) to teach TypeScript a new narrowing rule.

interface ApiSuccess { status: "success"; data: string; }
interface ApiError  { status: "error"; message: string; }
type ApiResponse = ApiSuccess | ApiError;

// Custom type guard with type predicate
function isSuccess(res: ApiResponse): res is ApiSuccess {
  return res.status === "success";
}

function handle(res: ApiResponse) {
  if (isSuccess(res)) {
    console.log(res.data); // ✅ res is ApiSuccess
  } else {
    console.log(res.message); // ✅ res is ApiError
  }
}

Q18 What is type narrowing in TypeScript?

Type narrowing is TypeScript's ability to refine a wide type to a narrower one within conditional blocks. When you perform a runtime check (like typeof x === "string"), TypeScript tracks this through the control flow and adjusts the type accordingly.

Built-in narrowing techniques:

  • typeof x === "string" — narrows primitives
  • x instanceof Error — narrows classes via prototype chain
  • "prop" in obj — narrows by property existence
  • x !== null / truthiness checks — eliminates nullish types
  • Discriminated unions — narrows by a shared literal field
function double(x: string | number): string | number {
  if (typeof x === "string") {
    return x.repeat(2); // ✅ x is string here
  }
  return x * 2; // ✅ x is number here
}
💡 Memory Hook: "Prove what type it is, then I'll let you use it." TypeScript won't let you call .toUpperCase() on string | number — but after typeof x === "string", it knows x is safe.

Q27 What are discriminated unions?

A discriminated union (also called a tagged union) is a pattern where each member of a union type shares a common property with a unique literal value. That shared property is the discriminant (commonly named kind, type, or status).

TypeScript uses the discriminant to automatically narrow the type in switch/if blocks. This is the go-to pattern for modeling state machines, network responses, and any data with mutually exclusive shapes. 

type Shape =
  | { kind: "circle"; radius: number }
  | { kind: "rect"; w: number; h: number }
  | { kind: "triangle"; base: number; height: number };

function area(s: Shape): number {
  switch (s.kind) {
    case "circle":
      return Math.PI * s.radius ** 2;
    case "rect":
      return s.w * s.h;
    case "triangle":
      return 0.5 * s.base * s.height;
  }
}
💡 Memory Hook: Discriminated union = tagged union. The discriminant (kind) is the tag that lets TypeScript narrow safely. Always use this for state machines — kind: "loading" | "success" | "error".

Q28 What are type guards?

Type guards are runtime checks that narrow types at runtime and inform the TypeScript compiler about the narrowed type. They come in several forms:

Built-in type guards: typeof, instanceof, and the in operator — these work out of the box.

Custom type guards use a type predicate (parameterName is Type) as the return type. This tells TypeScript: "If this function returns true, treat the argument as Type." 

// Custom type guard with type predicate
function isString(val: unknown): val is string {
  return typeof val === "string";
}

function process(input: unknown) {
  if (isString(input)) {
    console.log(input.toUpperCase()); // ✅ input is string
  }
}

Type predicates are especially useful for complex checks — e.g., validating API response shapes, checking if an object satisfies an interface, or validating parsed JSON.

💡 Memory Hook: Custom type guards with x is Type = "I promise this function checks the type correctly." You're teaching TypeScript a narrowing rule it can't figure out on its own.

🧪 Quick Check

1. Which operator narrows types by checking the prototype chain?

✅ Correct! instanceof checks the prototype chain and narrows the type accordingly — e.g., err instanceof Error.
❌ instanceof is the operator that checks the prototype chain. typeof checks primitives, in checks property existence, and as is a type assertion (not a runtime check).

2. What makes a union "discriminated"?

✅ Correct! A discriminated union shares a common property (the discriminant, e.g., kind) where each member has a unique literal value for that property.
❌ The defining feature is a shared property with a unique literal value per member — e.g., kind: "circle" vs kind: "rect". This enables type-safe narrowing in switch/if.

3. What does val is Type signify in a function return type?

✅ Correct! val is Type is a type predicate. When the guard function returns true, TypeScript narrows the parameter to Type in the calling scope.
❌ val is Type is a type predicate — it tells TypeScript: "If this function returns true, treat val as Type." It's not a cast or assertion; it's a narrowing guarantee.
