What is TypeScript?

TypeScript is a statically typed superset of JavaScript developed by Microsoft. It compiles to plain JavaScript — browsers and Node.js never run TypeScript directly. The compiler (tsc) checks your types at build time and strips them away, producing clean JS output.

The core value proposition: catch bugs before runtime. Instead of discovering undefined is not a function in production, TypeScript flags it in your editor as you type.

Beyond error checking, TypeScript enables powerful IDE features: autocompletion, inline documentation, safe refactoring (rename a property and every usage updates), and "go to definition" across your entire codebase.

1 What is TypeScript and why use it?

TypeScript is a statically typed superset of JavaScript that compiles to plain JS. It adds optional type annotations, compile-time error checking, better IDE support (autocompletion, refactoring), and improved code maintainability. Catches bugs before runtime.

💡 Memory Hook: TypeScript = JavaScript + types. Think of it as a spell-checker for your code — catches mistakes before you run it.

2 What is the difference between TypeScript and JavaScript?

JavaScript is dynamically typed — types are checked at runtime, meaning type errors surface as exceptions in production. TypeScript adds static typing — types are checked at compile time, catching errors during development. TS also adds interfaces, enums, generics, access modifiers, and advanced type features.

Key point: TS code compiles to JS — browsers don't run TypeScript directly. The type information exists only at compile time.

💡 Memory Hook: JS = types at runtime (surprises in production). TS = types at compile time (caught during development).

3 What are the basic types in TypeScript?

TypeScript has the following built-in types:

JavaScript primitives (carried over): string, number, boolean, null, undefined, bigint, symbol, object

TypeScript additions: any, unknown, void, never, Array<T>, tuple, enum 

let name: string = "Ali";
let age: number = 25;
let isActive: boolean = true;
let items: string[] = ["a", "b"];
💡 Memory Hook: Same as JS primitives + any, unknown, void, never, enum, tuple — the TypeScript-specific additions.

4 What is the any type and why should you avoid it?

any disables type checking entirely — you can assign anything to it and access any property without compiler errors. It defeats the entire purpose of using TypeScript. Every property access on any returns any, creating a cascading "type blackout" that spreads through your codebase.

The safe alternative: Use unknown when you don't know the type. With unknown, you can assign any value to it, but you must narrow the type (via typeof, instanceof, etc.) before using it. 

// ❌ any — no checks at all
let x: any = "hello";
x.nonExistentMethod(); // No error! Crashes at runtime

// ✅ unknown — must prove the type first
let y: unknown = "hello";
y.toUpperCase(); // Error! Must narrow first
if (typeof y === "string") y.toUpperCase(); // OK
💡 Memory Hook: any = "turn off TypeScript". unknown = "I don't know yet, but I'll check before using". Always prefer unknown.
