Quick Reference

TypeScript Type System

Primitives, type operators, literal types, and common patterns — all in one scannable page.

Primitive Types

JavaScript primitives, fully typed in TypeScript.

TypeDescriptionExample
string Textual data 
let name: string = "Alice";
number All numeric values (int + float) 
let age: number = 30;
let pi: number = 3.14;
boolean true or false 
let active: boolean = true;
null Intentional absence of value 
let empty: null = null;
undefined Variable declared but not assigned 
let x: undefined = undefined;
bigint Arbitrarily large integers 
let big: bigint = 9007199254740991n;
symbol Globally unique identifiers 
let sym: symbol = Symbol("id");
object Any non-primitive value 
let obj: object = { key: "val" };

TypeScript-Specific Types

Types that don't exist at runtime — purely for compile-time safety.

TypePurposeExample
any Opt out of type checking entirely. Avoid. 
let data: any = 42;
data = "now a string"; // OK, unsafe
unknown Safe alternative to any — requires narrowing 
let val: unknown = "hello";
if (typeof val === "string") {
  val.toUpperCase(); // OK after narrowing
}
void Function returns nothing 
function log(msg: string): void {
  console.log(msg);
}
never Unreachable / always throws 
function fail(msg: string): never {
  throw new Error(msg);
}
Array<T> Homogeneous list of T 
let nums: Array<number> = [1, 2, 3];
let names: string[] = ["a", "b"]; // shorthand
Tuple Fixed-length, fixed-type array 
let pair: [string, number] = ["age", 30];
let rgb: [number, number, number] = [255, 0, 128];
Enum Named constant set 
enum Direction {
  Up = "UP",
  Down = "DOWN",
}
let dir: Direction = Direction.Up;

Type Operators

The building blocks for composing types.

OperatorNameExample
| Union — value can be one of several types 
let id: string | number;
id = "abc"; // OK
id = 42;    // OK
& Intersection — combines all members 
type Named = { name: string };
type Aged = { age: number };
type Person = Named & Aged;
keyof Extract keys of a type as a union 
type Keys = keyof { x: number; y: number };
// "x" | "y"
typeof Capture the type of a value 
const config = { debug: true, port: 3000 };
type Config = typeof config;
as Type assertion (override inferred type) 
let val = "hello" as string | number;
as const Assert literal types + readonly 
const perms = ["read", "write"] as const;
// type: readonly ["read", "write"]
satisfies Validate against a type without widening 
const palette = {
  red: [255, 0, 0],
  green: "#00ff00",
} satisfies Record<string, string | number[]>;

Literal Types

Types that represent exact values — the narrowest possible type.

KindExample
String literal 
type Method = "GET" | "POST" | "DELETE";
let m: Method = "GET"; // only these 3 strings
Number literal 
type Port = 3000 | 8080 | 443;
let p: Port = 3000;
Boolean literal 
type Truth = true;
let t: Truth = true; // only true
Template literal 
type Event = `on${Capitalize<string>}`;
// "onClick", "onHover", etc.

type CSSValue = `${number}${"px" | "rem" | "em"}`;
let w: CSSValue = "16px";

Common Patterns

The patterns you'll use every day.

PatternSyntaxExample
Optional property prop?: T 
interface User {
  name: string;
  email?: string; // may be undefined
}
Readonly readonly 
interface Config {
  readonly apiKey: string;
}
// or
const arr: ReadonlyArray<number> = [1, 2];
Index signature [key: K]: V 
interface StringMap {
  [key: string]: string;
}
let dict: StringMap = { hello: "world" };
Function type (params) => return 
type Callback = (data: string) => void;
type Adder = (a: number, b: number) => number;
Generic function <T>(...) => ... 
function identity<T>(val: T): T {
  return val;
}
const r = identity("hi"); // type: string
Type guard (narrowing) typeof / in / instanceof 
function format(x: string | number) {
  if (typeof x === "string") {
    return x.padStart(10);
  }
  return x.toFixed(2);
}
