Glossary

TypeScript Glossary

Every term you'll encounter — defined, with code examples.

A

  • Ambient Declaration
    Tells the compiler about a value that exists at runtime but has no TypeScript source. Uses declare keyword. 
    declare const $: any; // jQuery loaded via CDN
declare module "some-lib" {
  export function doThing(): void;
}
  • Assertion
    Override the compiler's inferred type using as or angle-bracket syntax. Use sparingly — it silences the checker. 
    let val = "hello" as string | number;
let el = document.getElementById("x") as HTMLInputElement;

B

  • Brace-style
    TypeScript's structural typing means two types are compatible if their shape matches, regardless of the declaration form (interface vs. type alias vs. inline). 
    // These are structurally identical:
interface Point { x: number; y: number; }
type Pt = { x: number; y: number };
const p: Point = { x: 1, y: 2 } as Pt; // OK

C

  • Conditional Type
    A type that selects between two branches based on a constraint. Syntax: T extends U ? X : Y. 
    type IsString<T> = T extends string ? true : false;
type A = IsString<string>;  // true
type B = IsString<number>;  // false

D

  • Declaration File (.d.ts)
    A file containing only type declarations (no runtime code). Describes the shape of JavaScript libraries. 
    // utils.d.ts
export declare function greet(name: string): string;
  • Declaration Merging
    The compiler combines multiple declarations with the same name into a single definition. Common with namespaces and interfaces. 
    interface Box { height: number; }
interface Box { width: number; }
// Box now has both height and width
  • Discriminated Union
    A union where every member shares a common property (the discriminant) with a unique literal value. Enables exhaustive narrowing. 
    type Shape =
  | { kind: "circle"; radius: number }
  | { kind: "rect"; w: number; h: number };

function area(s: Shape): number {
  switch (s.kind) {
    case "circle": return Math.PI * s.radius ** 2;
    case "rect": return s.w * s.h;
  }
}

E

  • Enum
    A set of named constants. Prefer string enums or union literals for type safety. 
    enum Status {
  Active = "ACTIVE",
  Inactive = "INACTIVE",
}
let s: Status = Status.Active;
  • Exhaustiveness Check
    Ensures every case in a union is handled. Use never to get compiler errors if a case is missed. 
    function assertNever(x: never): never {
  throw new Error(`Unhandled: ${x}`);
}
function handle(s: Shape) {
  switch (s.kind) {
    case "circle": return;
    default: assertNever(s); // Error if new case added
  }
}

G

  • Generic
    A type variable that lets functions, interfaces, and classes operate on multiple types while preserving type safety. 
    function first<T>(arr: T[]): T | undefined {
  return arr[0];
}
const n = first([1, 2, 3]); // type: number | undefined

I

  • Index Signature
    Defines the type for dynamic keys on an object. Syntax: [key: K]: V. 
    interface Env {
  [key: string]: string | undefined;
}
const env: Env = { HOME: "/home/user" };
  • Index Type
    Using T[K] to look up the type of a property. keyof T gets all keys as a union. 
    function get<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}
  • Intersection Type
    Combines multiple types into one using &. The result has all members from each constituent. 
    type Timestamped = { createdAt: Date };
type User = { name: string } & Timestamped;
// { name: string; createdAt: Date }

L

  • Literal Type
    A type that represents exactly one value. String, number, and boolean literals are the most common. 
    type Direction = "left" | "right";
type Bit = 0 | 1;
type Yes = true;

M

  • Mapped Type
    Creates a new type by transforming every property of an existing type. Built-in utility types like Partial and Readonly are mapped types. 
    type Optional<T> = {
  [K in keyof T]?: T[K];
};
// Equivalent to Partial<T>

N

  • Namespace
    An organizational construct that groups related types, values, and functions. Largely replaced by ES modules but still used in .d.ts files. 
    namespace MathUtils {
  export function square(n: number): number {
    return n * n;
  }
}
const r = MathUtils.square(4); // 16
  • Narrowing
    The process of refining a type to a more specific one within a conditional branch. Achieved via typeof, instanceof, in, or discriminated unions. 
    function format(x: string | number) {
  if (typeof x === "string") {
    return x.toUpperCase(); // x: string
  }
  return x.toFixed(2); // x: number
}
  • Never
    The type of values that never occur. Used for unreachable code, exhaustive checks, and functions that always throw. 
    function fail(msg: string): never {
  throw new Error(msg);
}
type Empty = string & number; // never — impossible
  • Nominal Typing
    A type system where compatibility depends on name, not structure. TypeScript uses structural typing by default, but nominal patterns can be simulated. 
    type USD = number & { __brand: "USD" };
type EUR = number & { __brand: "EUR" };
// Can't accidentally mix USD and EUR
  • Nullish
    Values that are null or undefined. The nullish coalescing operator ?? and optional chaining ?. handle these. 
    const name = user?.name ?? "Anonymous";
// Falls back only for null/undefined, not ""

O

  • Optional
    A property or parameter that may be omitted. Denoted with ?. 
    interface Config {
  host: string;
  port?: number; // optional — can be undefined
}
function greet(name: string, title?: string) {}
  • Overload
    Multiple function signatures for the same function name. The implementation signature must be compatible with all overloads. 
    function parse(input: string): number;
function parse(input: number): string;
function parse(input: string | number) {
  return typeof input === "string"
    ? parseInt(input, 10)
    : String(input);
}

P

  • Predicate (Type Predicate)
    A function return type annotation that tells the compiler the argument has been narrowed. Syntax: paramName is Type. 
    function isString(val: unknown): val is string {
  return typeof val === "string";
}
if (isString(x)) {
  x.toUpperCase(); // TS knows x is string
}

R

  • Readonly
    Prevents reassignment of properties or array elements. Applied via readonly modifier or the Readonly<T> utility type. 
    interface Point {
  readonly x: number;
  readonly y: number;
}
const arr: ReadonlyArray<number> = [1, 2];
// arr.push(3); // Error

S

  • satisfies
    Validates an expression against a type without widening or losing the inferred literal types. Introduced in TS 4.9. 
    const palette = {
  red: [255, 0, 0],
  blue: "#0000ff",
} satisfies Record<string, string | number[]>;
// palette.red is still inferred as number[]
  • Structural Typing
    TypeScript's core type system — two types are compatible if they have the same shape, regardless of their names. Also called "duck typing." 
    interface Named { name: string; }
const user = { name: "Alice", age: 30 };
const n: Named = user; // OK — has a `name`

T

  • Template Literal Type
    Combines literal types using template syntax to create new string literal types. 
    type EventName = `on${Capitalize<string>}`;
type CSS = `${number}${"px" | "rem"}`;
  • Tuple
    An array with a fixed number of elements where each element has a known type (potentially different). 
    type Pair = [string, number];
const p: Pair = ["age", 25];
type StringNums = [string, ...number[]]; // rest element
  • Type Guard
    A runtime check that narrows a type within a scope. Built-in (typeof, instanceof, in) or custom (type predicates). 
    function isNonNull<T>(x: T | null): x is T {
  return x !== null;
}
  • Type Inference
    The compiler automatically determines types when annotations are omitted. Occurs for variables, return types, generics, and more. 
    let x = 42;    // inferred as number
const s = "hi"; // inferred as "hi" (literal)
function add(a: number, b: number) {
  return a + b; // return type inferred as number
}
  • Type Literal
    An anonymous object type written inline — similar to an interface but without a name. 
    function draw(opts: { color: string; size: number }) {
  // opts is a type literal
}
  • Type Variable
    A placeholder type (like <T>) in generic functions, interfaces, or classes. Gets bound to a concrete type at each call site. 
    function identity<T>(val: T): T {
  return val; // T is the type variable
}

U

  • Union Type
    A type that can be one of several types, joined with |. 
    let id: string | number;
id = "abc"; // OK
id = 42;    // OK
  • unknown
    The type-safe counterpart to any. Accepts any value but requires narrowing before use. 
    let x: unknown = JSON.parse(input);
// x.foo();     // Error — not narrowed
if (typeof x === "string") {
  x.toUpperCase(); // OK
}

V

  • Variadic
    Related to variable-length argument lists. TypeScript handles these via rest parameters and tuple spread types. 
    function log(...args: unknown[]): void {
  console.log(...args);
}
type Tail<T extends any[]> = T extends [infer _, ...infer Rest]
  ? Rest : never;
  • void
    The return type for functions that don't return a value. Different from undefined — it explicitly means "no return." 
    function log(msg: string): void {
  console.log(msg);
}
