React Glossary

Every term you'll encounter — defined, with an interview-ready one-liner.

B

  • Batching
    React collapsing multiple state updates into a single re-render for performance. React 18 introduced automatic batching — updates in promises, timeouts, and native handlers batch too, not just React event handlers. "React 18 batches updates everywhere — three setState calls in a handler produce one render."

C

  • Cleanup
    The optional function returned from useEffect that runs before the next effect and on unmount — used to tear down subscriptions, clear timers, cancel fetches. "Cleanup runs before the next effect and on unmount — missing it is the #1 source of memory leaks and stale-state bugs."
  • Compound Components
    A set of components that share state implicitly through context, exposing a composable API — like HTML's <select> + <option>. Used by design systems (Radix, Headless UI) for Tabs, Accordion, Menu. "Like <select> + <option> — children share parent state via context, no explicit props."
  • Concurrent Rendering
    React 18's rendering model where updates can be interrupted, paused, and resumed, allowing urgent updates to preempt non-urgent ones. Powered by Fiber; exposed via startTransition, useTransition, useDeferredValue. "Concurrent rendering lets React interrupt a slow render to handle an urgent update — the foundation of transitions and Suspense for data."
  • Controlled Component
    An input whose value is driven by React state — value mirrors state, onChange updates it. React is the single source of truth. "Controlled = React drives; state is the source of truth. Use it for live validation and formatting."

E

  • Effect
    Code run by useEffect that synchronizes a component with an external system (network, DOM, subscription). Effects run after render (after paint). "An effect synchronizes the component with something outside React. Don't use it to chain state — derive during render where possible."
  • Error Boundary
    A class component that catches errors thrown during rendering, in lifecycle methods, and in constructors of its children — displaying fallback UI. Must be a class component. "try/catch for JSX — but only for render-phase errors, not event handlers or async code."

F

  • Fiber
    React's reconciliation engine (rewritten in v16). Breaks rendering work into interruptible units ("fibers"), supports priorities, and is the foundation for concurrent features. "Fiber = React 16's engine. Interruptible, priority-aware work units — the basis of transitions and concurrent rendering."

H

  • Higher-Order Component (HOC)
    A function that takes a component and returns an enhanced component. The pre-hooks pattern for cross-cutting concerns (auth, logging). Largely replaced by hooks. "HOC = wrapper function: component in, enhanced component out. Hooks replaced most uses; connect() in redux is the famous example."
  • Hydration
    The process where React "adopts" server-rendered HTML on the client — attaching event listeners and reconciling the existing DOM rather than rebuilding it. Uses hydrateRoot. "Hydration wakes up server HTML with JS. Mismatches between server and client output cause warnings and re-renders."

M

  • Memoization
    Caching a computed result or function reference so it isn't recreated on every render. Provided by useMemo (values), useCallback (functions), and React.memo (components). "Memoization caches work — but it isn't free. Reach for it only when profiling shows a real bottleneck, or when identity stability is required."

O

  • Optimistic Update
    Showing a UI change immediately, before the server confirms it, and reverting if it fails. React 19 provides useOptimistic for this pattern. "Assume the action succeeds, show it instantly, revert on failure — useOptimistic handles the bookkeeping."

P

  • Portal
    A way to render children into a DOM node outside the parent's hierarchy, while keeping them in the React tree (so events bubble and context works). Uses createPortal. "Portal = render here, show there. Modals stay in the React tree but appear at document.body."
  • Prop Drilling
    Passing props through intermediate components that don't use them, just to reach a deeply nested child. A code smell when 3+ intermediates are pure forwarders. "Passing the baton through people who don't need it. Fix with composition, Context, or a state library."
  • Progressive Enhancement
    Building UI so it works at a baseline without JavaScript, then enhancing with JS. React Server Actions enable this — forms POST natively, JS adds interactivity. "Server Actions work without JS — the form POSTs natively; JS just enhances the experience."

R

  • Reconciliation
    React's algorithm for diffing old vs new virtual DOM trees to determine the minimum real-DOM mutations. Rules: different type → rebuild; same type → update in place; children matched by key. "Reconciliation = 'what changed?' Different type = nuke and rebuild; same type = update in place; keys = identity for list children."
  • Render Props
    A prop whose value is a function returning JSX — the parent owns state and calls the function to let the consumer decide rendering. Largely replaced by hooks. "Render prop = 'you tell me how to render, I give you the data.' Hooks replaced it; you'll see it in older libraries."

S

  • SSR (Server-Side Rendering)
    Rendering React to HTML on the server and sending it to the client for fast first paint and SEO. The client then hydrates it into an interactive app. "SSR = render to HTML on the server for fast paint + SEO, then hydrate on the client for interactivity."
  • RSC (React Server Components)
    Components that run only on the server, can be async, and ship zero JavaScript to the client. Client Components opt in with "use client". "RSC = zero-JS backend components. They fetch data directly, ship no code, and pair with Client Components for interactivity."
  • Stale Closure
    A bug where a long-lived function (interval, subscription) captures variables from the render it was created in, never seeing updates. Fixed by adding deps, using a ref, or the functional updater. "A stale closure captures old state forever. Fix: add the dep, use a ref, or use setX(prev => ...)."
  • StrictMode
    A dev-only wrapper that surfaces potential problems — unsafe lifecycles, deprecated APIs — and double-invokes render and effects to expose impure code. Zero production effect. "StrictMode double-invokes in dev to catch impure code. If your effect breaks when run twice, it isn't pure."
  • Suspense
    A component that shows a fallback while its children are loading (lazy components, suspending data fetches via use). Pairs with Error Boundaries for failure states. "<Suspense fallback={...}> shows a spinner while children load — code splitting and the React 19 data model both rely on it."

T

  • Transition
    A low-priority state update marked by startTransition or useTransition. Urgent updates (typing) can interrupt it. isPending flags in-progress work. "A transition marks an update as non-urgent — urgent input stays snappy; isPending gives you a loading flag."
  • Tearing
    An inconsistency where different parts of the UI read different snapshots of an external store during a single concurrent render pass. useSyncExternalStore prevents it. "Tearing = different UI sections showing different snapshots. useSyncExternalStore is the cure for external stores under concurrent rendering."

U

  • Uncontrolled Component
    An input where the DOM owns the value; React reads it on demand via a ref (defaultValue sets the initial). Good for simple forms and non-React integrations. "Uncontrolled = DOM drives; read via ref at submit. Less code, less control — good for simple forms or legacy widgets."

V

  • Virtual DOM
    A lightweight in-memory JS object tree that mirrors the real DOM. React diffs old vs new virtual trees and applies only the minimal changes to the real DOM (reconciliation). "Virtual DOM = a blueprint. React diffs blueprints cheaply, then touches the real DOM as little as possible."
