What is React?

React is a JavaScript library for building user interfaces, created at Meta (Facebook) and open-sourced in 2013. The key idea: instead of imperatively poking at the DOM (find element, update its text, add a class), you describe the UI as a function of state, and React figures out which DOM changes to make. This is the "UI = f(state)" model.

React focuses on the view layer — it is a library, not a full framework. You bring your own routing (React Router, TanStack Router), state management (Context, Zustand, Redux), and build tooling (Vite, Next.js). This "unopinionated" nature is both its flexibility and its interview relevance: you're expected to know the surrounding ecosystem and the trade-offs.

Modern React (18/19) is built on a concurrent renderer and a reconciliation engine called Fiber. We'll dig into those internals in Lesson 8. For now, the foundation: components return JSX, state drives re-renders, and React keeps a virtual DOM to diff efficiently.

1 What is React?

React is a JavaScript library for building user interfaces using reusable components. It uses a virtual DOM for efficient updates, JSX for templating, and a unidirectional (top-down) data flow. Created by Meta (Facebook).

The core value proposition is the declarative model: you describe what the UI should look like for a given state, and React handles updating the DOM when state changes. Contrast this with jQuery-era imperative code, where you manually found elements and mutated them. 

// Declarative: UI is a function of state
function Counter({ count }) {
  return <button>Clicked {count} times</button>;
}
💡 Memory Hook: React = component-based UI library (not a framework). It owns the view layer — you add routing, state, and tooling yourself. Interview one-liner: "a declarative, component-based UI library that renders UI as a function of state."

2 What is JSX?

JSX is a syntax extension that lets you write HTML-like code inside JavaScript: const el = <h1>Hello</h1>. A compiler (Babel or the modern JSX transform) turns it into React.createElement() (or the automatic jsx()) calls. JSX is not HTML — it's syntactic sugar for JavaScript function calls.

Because it compiles to plain JS, you can embed any JavaScript expression inside it with { }. Differences from HTML: class becomes className, for becomes htmlFor, attributes are camelCase (onClick not onclick), and style takes an object not a string. 

// JSX...
const el = <h1 className="title">Hello {name}</h1>;

// ...compiles to:
const el = jsx("h1", { className: "title", children: ["Hello ", name] });
💡 Memory Hook: JSX = HTML in JavaScript (not JavaScript in HTML). It's compile-time sugar for function calls — browsers never see JSX. Remember the gotchas: className, htmlFor, one root element per return (or use a Fragment).

3 What is the Virtual DOM?

The Virtual DOM is a lightweight JavaScript object representation of the real DOM. React keeps an in-memory tree that mirrors the rendered DOM. On every state change, React builds a new virtual tree, compares (diffs) it against the previous one, and computes the minimum set of real DOM mutations required. This comparison process is called reconciliation.

Why is this fast? Real DOM operations (layout, paint, reflow) are expensive. By diffing cheap JavaScript objects first, React batches and minimizes the expensive real-DOM writes. Note: the virtual DOM is not magic — its win is predictable, batched, minimal updates, not raw speed versus a perfectly hand-optimized update. 

// A virtual DOM node is just an object:
{
  type: "div",
  props: { className: "card", children: "Hi" }
}
💡 Memory Hook: Virtual DOM = a blueprint. React compares old and new blueprints, then applies only the diffs to the real DOM. Much cheaper than rewriting everything. The diffing is reconciliation (see Lesson 8).

4 What are components in React?

Components are reusable, self-contained UI building blocks. There are two kinds:

  • Function components — plain functions that return JSX. The modern standard, paired with hooks for state and effects.
  • Class components — ES6 classes with a render() method and lifecycle methods. Legacy, but you must still recognize them (and know they're the only way to write an error boundary — see Lesson 6).

A component's name must start with a capital letter — lowercase tags are treated as DOM elements, capitalised tags as components. Modern React almost exclusively uses function components with hooks. 

// Function component (modern)
function Greeting({ name }) {
  return <h1>Hello, {name}</h1>;
}
💡 Memory Hook: Function components = modern way; class components = legacy (but still needed for error boundaries). Capitalized names = components; lowercase = HTML elements.

5 What are props in React?

Props (properties) are read-only data passed from a parent component to a child. Data flows one direction — top-down (parent → child). A child cannot modify its props; they are immutable. To communicate upward, the parent passes a callback function as a prop, and the child calls it.

This unidirectional flow makes data flow predictable and debuggable — you can always trace where data came from by walking up the tree. Contrast this with two-way binding (Angular, Vue) where changes propagate in both directions. 

// Parent passes props down + a callback for upward comm
<Parent>
  <Child name="Ali" onDone={handleDone} />
</Parent>

// Child: props are read-only
function Child({ name, onDone }) {
  return <button onClick={onDone}>Hi {name}</button>;
}
💡 Memory Hook: Props = arguments for components. One-way street: parent → child. To go upward, the parent hands down a callback the child invokes. Props are immutable — mutating them is a React error.

🧪 Quick Check

1. What does JSX compile to at build time?

✅ Correct! JSX is syntactic sugar — the compiler turns <h1>Hi</h1> into a createElement/jsx() call. Browsers never see JSX.
❌ JSX is sugar over function calls. <h1>Hi</h1> becomes something like jsx("h1", { children: "Hi" }).

2. Which is true about the virtual DOM?

✅ Correct! The virtual DOM is a lightweight JS representation. React diffs old vs new, then applies only the minimal real-DOM changes (reconciliation).
❌ The virtual DOM is an in-memory JS tree React diffs first. It's not magic — its win is predictable, minimal, batched real-DOM updates.

3. How does a child component send data back up to its parent?

✅ Correct! Data flows down via props; upward communication uses a callback the parent hands down. Props are immutable to the child.
❌ Props are read-only and flow only downward. The parent must pass a callback prop that the child invokes to communicate upward.
